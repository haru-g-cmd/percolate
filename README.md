# Percolate

Infrastructure resilience testing platform that lets you model distributed systems as graphs and simulate cascading failures through them in real time. You build a topology where nodes are services, databases, caches, load balancers, etc. and edges are the dependencies between them, then inject failures like crashes, memory leaks, CPU saturation, or network partitions into individual nodes and watch the damage propagate through the system. There are four prebuilt topologies (e-commerce, social media, Kubernetes cluster, ML pipeline) with 16 incident scenarios like database meltdowns and cache stampedes. You can toggle resilience patterns (circuit breakers, retries, fallbacks, redundancy, rate limiting) on any node to see how they reduce the blast radius of a failure. The dashboard tracks health, error rates, throughput, latency, MTTR/MTTF, and a weighted resilience score, and it generates recommendations like identifying single points of failure or missing circuit breakers on critical nodes.

The backend is a Node.js/Express server with WebSocket connections to push simulation state in real time. Each WebSocket connection gets its own simulation engine instance, so multiple users can run independent simulations. The simulation engine runs a tick-based loop with three failure propagation models: immediate (full impact on injection), gradual (steady degradation with randomized jitter), and threshold (slow degradation for 15 ticks then exponential collapse, mimicking things like connection pool exhaustion). Failures cascade through the dependency graph in both directions, upstream through edges weighted by severity and spread rate, and downstream at a reduced rate when a source node drops below 50% health. Resilience patterns apply multiplicative damage reduction but are capped so at least 20% of impact always gets through. Recovery is suppressed on any node that still has a critically failing dependency. The whole state, including per node metrics, blast radius, and auto generated recommendations, gets pushed to the client every tick over the WebSocket. Simulation results can be saved to a MySQL database (8 tables covering topologies, nodes, edges, failure modes, simulations, events, metrics snapshots, and recommendations) for later review. All of the topology and scenario data is seeded from SQL scripts that run on first startup.

The React client renders the topology as a D3.js force directed graph with SVG nodes showing health arcs, status indicators, and shockwave effects on failure injection. Recharts handles the real time metrics charts (health, errors, throughput as stacked area charts), and Zustand manages all client-side state including a ring buffer of 150 metrics snapshots. The UI has a topology selector, play/pause/reset controls, a speed slider (0.25x to 4x), a chaos mode toggle for random failure injection, node detail panels with resilience pattern toggles and failure injection buttons, scenario cards, an analysis panel with resilience score breakdowns and recommendations, and an auto scrolling event feed. The whole thing is containerized with Docker Compose (multi stage build for the React client, MySQL 8.0 with health checks and automatic schema seeding).

```bash
docker compose up --build
```

Opens on `localhost:3000`. Without Docker, set up MySQL 8.0, create a `.env` with your DB credentials, run `npm run db:init` to seed the schema, build the client with `cd client && npm install && npm run build`, then `npm start`.
